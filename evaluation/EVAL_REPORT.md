# Router 意图识别评测报告（测量 → 改进）

> 本报告记录 Router 意图识别的离线评测过程与优化效果，数据来自 `evaluation/` 下的评测结果文件。
> 评测集：100 条用例（`router_eval_cases.json`），覆盖天气查询、火车票查询、组合查询、缺失参数、否定表达等场景。

## 各阶段指标对比

| 指标 | Baseline（无思考） | Thinking（开启思考模式） | 当前（思考 + 规则纠偏 + 槽位推导） |
| --- | --- | --- | --- |
| 失败用例数 | 27 / 100 | 3 / 100 | 1 / 100 |
| 意图识别准确率 | 90% | 97% | 99% |
| Agent 路由准确率 | 90% | 98% | 99% |
| 槽位精确匹配 | 82% | 99% | 99% |
| 缺失槽位判断准确率 | 75% | 100% | 100% |
| 追问判断准确率 | 79% | 100% | 100% |
| 平均单用例延迟 | 1.4s | 5.5s | 5～7s（随机器负载波动） |

对应数据文件：
- `router_eval_results_100_baseline.json`
- `router_eval_results_100_thinking.json`
- `router_eval_results_100_post_reliability.json`

## 优化做了什么

1. **开启模型思考模式（thinking）**：让 Qwen 在输出 JSON 前先推理，显著提升意图与槽位判断的准确率。
2. **在 `analyzer` 中加入规则纠偏**：用关键词与否定表达模式（如“不要查天气，只查火车票”）修正模型输出，避免明显的路由错误；并补全火车票关键词（如“车次”）。
3. **确定性槽位推导**：组合查询中天气槽位可由票务槽位推出——缺 `city` 时用 `arrival_city`、缺 `fx_date` 时用 `travel_date`，不完全依赖 LLM 自填。
4. **可靠性加固**：超时、重试、JSON 解析鲁棒性等改动在保持准确率的同时增强了稳定性。

## 关键结论

- **路由失败率从 27% 降至 1%**（↓ 96%）。
- 意图识别 90% → 99%，Agent 路由 90% → 99%，缺失槽位判断 75% → 100%，追问判断 79% → 100%。
- 唯一残留失败是“做个简单出行查询”这类**不含任何领域关键词的模糊请求**，属于语义上本就两可的边界情形。
- **代价是延迟**：思考模式使单用例延迟由约 1.4s 升至约 5～7s。这是一个明确的“准确率 ↔ 延迟”权衡，实际部署可按场景取舍。

## 方法论要点（面试可讲）

- 先用评测集**量化**基线、**定位**失败模式，再有针对性地优化，而不是凭感觉调 prompt。
- 优化后**回归测试**，确认整体提升且无明显退化。
- 对引入的代价（延迟）做了**显式权衡分析**，而非只报喜不报忧。

## 简历 bullet（可直接使用）

> 构建 100 用例离线评测框架，量化多智能体路由的意图识别、Agent 选择与槽位抽取准确率及延迟；
> 通过开启模型思考模式、引入关键词/否定表达纠偏规则与确定性槽位推导，将路由失败率从 27% 降至 1%、意图识别准确率从 90% 提升至 99%，
> 并量化了思考模式带来的延迟权衡（1.4s → 5～7s）。
